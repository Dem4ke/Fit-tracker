
# 1 Введение

<p>
В данном документе дано описание архитектурной структуры и основных принятых<br/>
решений при проектировании системы слежения за здоровьем.
</p>
<p>
Fit-tracker является клиент-серверным многопользовательским приложением,<br/>
позволяющим отслеживать и записывать различные показатели, связанные с питанием и здоровьем   
</p>
<p>
Программа создана с помощью библиотеки Qt 6, с использованием декларативного языка QML<br/>
для создания пользовательского графического интерфейса. 
</p>
<p>
Все диаграммы созданы в ПО Drawio.<br/>
Все диаграммы можно посмотреть в папке Diagrams.
</p>
<p>
Краткие описания классов, и их полей даны в заголовочных (.h) файлах, <br/>
описания методов - в файлах с описанием (.cpp) над каждым из них.
</p>
<p>
<strong>Форматирование кода:</strong><br/>

- Приватные поля имеют суффикс "_", для обозначения данных переменных,<br/>
как инкапсулированных в классе, поля структур - без суффикса, как локальные переменные;
- Названия методов, функций и полей используют <em>camelStyle</em> - <br/>
начинаются с маленькой буквы и пишутся слитно;
- Все классы, описывающие диалоговые окна имеют суффикс <strong>"Dlg"</strong>;
- Название <em>enum</em> классов начинаются с заглавной литеры <strong>"E"</strong>.
</p>
<p>
<strong>Система сборки</strong><br/>
В качестве автоматической системы сборки, используется <em>CMake</em>.
</p>

# 2 Настройка окружения

<p>
Версия библиотеки Qt - 6.10.2 и выше <br/> 
В Qt Creator IDE все что нужно - это открыть основной CMakeLists файл <br/>
</p>
<p>
Во время создания системы использовалось IDE CLion от JetBrains. <br/>
В данной IDE отсутствует дизайнер. <br/>
Создать .ui файл возможно только в Qt Creator, однако не рекомендуется использование <br/>
таких интерфейсов из-за их громоздкости и неудобства настройки. <br/>
Настройка зависимостей в CLion: <br/>

- Все необходимые зависимости устанавливаются в Settings/Build/Cmake
- Установить путь до установленной библиотеки Qt <br/>
-DCMAKE_PREFIX_PATH="C:/Soft/Qt/6.10.2/mingw_64", где в кавычках указан путь
- Установить автоматическое применение windeployqt <br/>
-DDEV_DEPLOY=ON
</p>
<p>
<strong>Важно про deploy:</strong><br/>
Для грамотного отображения всех элементов приложения папки Config и Languages<br/> 
должны быть в директории сборки (в одной директории с исполняемым файлом)
</p>

# 3 Архитектура системы

<p>
Вся система делится на несколько основных модулей, выделенных в отдельные библиотеки:<br/>

- Библиотеку пользовательского графического интерфейса;
- Библиотеку основного ядра системы (операционный уровень);
- Библиотеку предметной области системы (уровень предметной области);

</p>
<p>
Каждая из этих библиотек рассмотрена подробно в своем разделе.<br/>
Также отдельно рассмотрена архитектура базы данных.
</p>

# 6 Библиотека предметной области системы

<p>
В данной библиотеке представлены Domain model level классы, представляющие модель предметной области, <br/>
связанную с бизнес-логикой предметной области.<br/>
<strong>Данная библиотека является общей для серверного приложения и для клиентской части.</strong>
</p>
<p>
Сущности, используемые для хранения и представления данных:<br/>

- Day (день, представляющий один день в календаре, для каждой даты уникален);
- Meal (прием пищи, представляющий данные об одном конкретном приеме пищи, не уникален)
- Food (еда, представляет собой данные по одному конкретному элементу пищи, может быть составным)
</p>

## 6.1 Day

<p>
Класс, описывающий день. Содержит в себе свои атрибуты, а также может включать в себя любое количество<br/>
приемов пищи (Meal)
</p>
<p>
Атрибуты:<br/>

- Уникальный идентификатор, по которому происходит обращение в базу данных (использует тип unsigned long long или quint64).
- Количество употребленной воды за день, в миллилитрах (тип int).
- Вес, в килограммах (тип float).
- Календарная дата, которую представляет объект этого класса (тип QDate).
- Обмеры тела (тип BodyMeasurements (п 6.1.1)).
- Максимальные значения по питательным веществам:<br/>
  - Применяется для отображения ограничений по количеству питательных веществ на дню. (тип SummaryNutrients (п 6.1.2)).
- Заметка пользователя (тип QString).
- Массив приемов пищи (тип Meal).
</p>

### 6.1.1 BodyMeasurements

<p>
Структура, хранящая в себе данные о замерах тела (объемы рук, ног, талии, бедер и тд).
</p>

### 6.1.2 SummaryNutrients

<p>
Структура, хранящая в себе общие значения питательных веществ.<br/>
</p>

## 6.2 Meal

<p>
Класс, описывающий прием пищи. Может включать в себя любое количество блюд (Food).
</p>
<p>
Атрибуты:<br/>

- Уникальный идентификатор, по которому происходит обращение в базу данных (использует тип unsigned long long или quint64).
- Название (тип QString).
- Массив блюд (тип Food).
- Общее значение питательных веществ за прием пищи (вычислимое значение, тип SummaryNutrients (п 6.1.2)).
</p>

## 6.3 IFood

<p>
Интерфейс, представляющий блюдо. Также может быть скастован в RecipeFood, то есть составное блюдо.
</p>

### 6.3.1 RecipeFood

<p>
Класс, описывающий составное блюдо. То есть имеет потомков, которые могут быть как SimpleFood, так и составным RecipeFood.
</p>
<p>
Атрибуты:<br/>

- Уникальный идентификатор, по которому происходит обращение в базу данных (использует тип unsigned long long или quint64).
- Атрибуты блюда (тип FoodAttributes (п 6.3.3)).
- Атрибуты рецепта (тип RecipeAttributes (п.6.3.4)).
- Массив блюд, из которых состоит рецепт (тип IFood).
</p>

### 6.3.2 SimpleFood

<p>
Класс, описывающий простое блюдо. Он может быть как отдельным блюдом, состоящим из одного ингридиента,<br/>
так и ингридиентом для составного блюда.
</p>
<p>
Атрибуты:<br/>

- Уникальный идентификатор, по которому происходит обращение в базу данных (использует тип unsigned long long или quint64).
- Атрибуты блюда (тип FoodAttributes (п 6.3.3)).
</p>

### 6.3.3 FoodAttributes

<p>
Класс, описывающий общие атрибуты любого блюда.<br/>
Включает в себя:<br/>

- Название (тип QString).
- Количество жиров (тип float).
- Количество белков (тип float).
- Количество углеводов (тип float).
- Вес блюда в граммах (тип float).
- Количество калорий (вычисляемое значение) (тип float).
</p>

### 6.3.4 RecipeAttributes

<p>
Класс, описывающий атрибуты рецепта.<br/>
Включает в себя:<br/>

- Количество порций (тип int).
- Время подготовки в минутах (тип float).
- Время готовки в минутах (тип float).
- Описание шагов приготовления (тип QString).
</p>

# 8 База данных

<p>

## 8.1 Таблица пользователей (users)

<p>
Таблица, описывающая пользователей системы.<br/>
Включает в себя:<br/>

- Уникальный идентификатор (тип BIGSERIAL).
- Имя пользователя (тип VARCHAR(255)).
- Электронная почта пользователя (тип VARCHAR(255)).
- Хэш пароля (тип TEXT).
- Заметка пользователя (тип TEXT).
</p>
